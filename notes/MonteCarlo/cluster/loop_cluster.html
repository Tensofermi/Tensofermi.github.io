Loop 更新¶

loop 更新是一类非局域更新。它不直接改变单个自由度，而是先在构型中识别闭合路径，再整体翻转、重连或改变这条路径上的状态。

在许多量子 Monte Carlo 和几何表象中，构型必须满足局域守恒或顶点约束。单点更新很容易破坏约束，或者只能在很小范围内移动缺陷。loop 更新的好处是：闭合 loop 的整体变化天然保持约束，同时能改变大尺度结构。

直观地说：

  1. 根据当前构型和模型权重生成可更新的图形元素。
  2. 从某个位置出发沿规则追踪，直到形成闭合 loop。
  3. 按满足平衡条件的概率翻转或重连整条 loop。

loop 更新与 cluster 更新有相似的精神：都试图把强关联的一组自由度作为整体更新，以降低自关联时间。区别是 cluster 常用于自旋团簇，loop 更常用于有守恒流、世界线或顶点约束的构型。

具体模型中的 loop 规则差异很大，本页只作为概念入口；详细推导应结合具体哈密顿量和图形表象来写。

约束构型中的困难¶

许多模型的构型空间带有局域约束。例如六顶点模型、dimer 模型、SSE 表象中的顶点构型、世界线构型中的粒子数守恒，都要求每个局域单元满足某种流入流出关系。若只更新一个局域自由度，通常会破坏约束。

以世界线为例，粒子数守恒意味着世界线不能凭空中断。若在某个虚时点单独改变占据数，构型会出现一个缺口。为了保持构型合法，更新必须沿着世界线继续传播这个变化，直到它闭合或被另一个缺陷吸收。loop 更新的精神就是把这件事系统化：沿着局域规则追踪一条路径，然后整体更新。

在图形表象中，构型权重常写成局域顶点权重的乘积：

\[ W(C)=\prod_v w_v(C_v). \]

其中 \(v\) 是顶点或局域单元，\(C_v\) 是该顶点附近的局域状态。loop 更新会改变一串顶点上的局域状态：

\[ C_v\to C_v', \]

并要求每个改变后的顶点仍然满足约束。

顶点与路径¶

loop 的构造通常需要先把模型写成顶点语言。每个顶点有若干条入射 leg，leg 上带有局域状态。例如自旋模型中可以用

\[ \uparrow,\downarrow \]

表示 leg 状态；玻色世界线中可以用粒子数或流量表示。

一次 loop 更新可以理解为：

  1. 随机选择一个起始 leg。
  2. 改变该 leg 的状态。
  3. 为了保持当前顶点合法，必须选择另一个 leg 一起改变。
  4. 从被选中的出口 leg 进入相邻顶点。
  5. 重复这个过程，直到回到起点形成闭合 loop。

如果每个顶点的入口到出口选择概率记为

\[ P_v(a\to b), \]

那么整条 loop 的 proposal probability 是沿途局域概率的乘积：

\[ A(C\to C')=\prod_{v\in {\rm loop}}P_v(a_v\to b_v). \]

为了让算法正确，需要让 proposal 和接受率共同满足 detailed balance。

局域平衡条件¶

一种优雅做法是直接在每个顶点上满足局域平衡。设顶点更新前后的局域状态为 \(C_v\) 和 \(C_v'\)，要求

\[ w_v(C_v)P_v(C_v\to C_v') = w_v(C_v')P_v(C_v'\to C_v). \]

若沿 loop 上每个顶点都满足这个关系，把所有顶点相乘就得到整条 loop 的平衡条件：

\[ W(C)A(C\to C') = W(C')A(C'\to C). \]

此时 loop 翻转可以接受率取为 \(1\)。这也是许多 directed loop 或 worm-like loop 算法高效的原因：把接受率问题拆成了局域散射概率的设计问题。

若局域概率没有完全吸收权重比，也可以先生成 loop，再用

\[ P_{\rm acc}(C\to C') = \min\left[ 1, \frac{W(C')A(C'\to C)} {W(C)A(C\to C')} \right] \]

接受或拒绝。

一个简单图像¶

考虑一个守恒流模型。每条边上有流量，顶点要求流入等于流出：

\[ \sum_{\rm in} j = \sum_{\rm out} j. \]

若某条边上的流量增加 \(1\)，相邻顶点会出现一个不平衡。为了修复它，需要从这个顶点选择另一条边也改变 \(1\)，不平衡就被传递到下一个顶点。这样不断传递，直到回到起点时，所有顶点重新满足约束。

闭合 loop 的整体变化可以写成

\[ j_e\to j_e+\Delta j_e, \]

其中只有 loop 上的边有 \(\Delta j_e\ne 0\)。由于每个顶点有一条进入变化和一条离开变化，局域守恒自动成立。

与 Worm 更新的关系¶

loop 更新和 worm 更新关系很近。loop 更新坚持构型始终在配分函数空间 \(\mathcal Z\) 中，路径最后闭合；worm 更新允许中间过程进入带有两个端点的格林函数空间 \(\mathcal G\)。

可以粗略理解为：

  • loop 更新：创建一条闭合路径，一次性翻转闭合结构。
  • worm 更新：创建两个端点，让端点在构型中移动，最后湮灭。

两者都利用了“沿约束传播更新”的思想。worm 更新更灵活，常用于测量 Green 函数；loop 更新结构更干净，常用于顶点模型或 SSE 中的非局域翻转。

算法流程¶

一个典型 loop 更新可以写成：

  1. 从当前合法构型 \(C\) 出发。
  2. 随机选一个起始 leg。
  3. 按局域散射概率选择出口 leg，并改变顶点状态。
  4. 沿着连接关系进入下一个顶点。
  5. 重复散射，直到路径闭合。
  6. 根据设计选择直接接受，或用 Metropolis 因子接受。
  7. 更新构型中的所有受影响顶点和边。

程序中需要维护的对象通常包括：

  • 顶点列表。
  • 每个顶点的 leg 状态。
  • leg 到相邻 leg 的连接关系。
  • 局域权重表 \(w_v\)。
  • 入口到出口的散射概率表。

常见问题¶

  • bounce 太多：入口 leg 又从原路返回，会让 loop 很短，降低效率。directed loop 方程常用来减少 bounce。
  • loop 过长：接近临界点时 loop 可能跨越系统，更新很有效，但一次更新代价也会变大。
  • 拓扑扇区：某些局域 loop 无法改变 winding number，需要允许 wrapping loop 或加入额外全局更新。
  • 权重为零的顶点：散射规则必须避免走进非法局域状态。
  • 详细平衡检查：对一个小系统枚举构型，检查 proposal 与权重比，是调试 loop 更新最有效的方法之一。

loop 更新的关键在于：把合法构型之间的非局域跳跃，拆成一串局域、可检查、满足平衡条件的散射过程。
